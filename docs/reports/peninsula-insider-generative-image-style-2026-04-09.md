## 1. Purpose

This document defines how Peninsula Insider should use generative imagery **if and when** it chooses to. It exists to solve a narrow problem: some V2 surfaces will need supporting visuals before commissioned photography exists, and the honest alternatives are either:

1. a generic stock image that weakens the brand, or
2. the existing CSS fog / gradient placeholder.

Generative imagery is acceptable only when it is **better than stock, clearly non-documentary, and tightly governed.** It is not a substitute for real photography of real Peninsula places.

The role of generative imagery at Peninsula Insider is therefore:

> **atmospheric illustration for non-specific editorial support slots, never documentary stand-in.**

That single sentence is the rule everything else in this document protects.

---

## 2. The core principle

Peninsula Insider is trying to become a premium editorial destination guide, not a moodboard. The brand's authority comes from naming real places, making real recommendations, and eventually showing real photography of the region. Generative imagery can support that brand only when it behaves like **editorial illustration** rather than fake photography.

If a viewer could reasonably mistake a generated image for evidence that a place actually looks like that, the image is in the wrong role.

So the working model is not:
- "make fake travel photos"

It is:
- "make controlled, filmic, place-adjacent illustrations that hold the page without pretending to prove anything."

## 3. Approved use cases

Generative imagery is approved only for the following slot types.

### 3.1 Approved

1. **Section-support textures and atmospherics**
   - newsletter block backgrounds
   - quote-break backgrounds
   - section dividers
   - subtle interstitial editorial art

2. **Non-specific editorial mood covers**
   - a seasonal journal package opener, where the image is clearly conceptual rather than reportage
   - an "autumn on the Peninsula" package where the image reads as atmosphere, not a named place

3. **Planner / utility surfaces that need rhythm**
   - itinerary headers
   - fallback visuals for filters, taxonomies, mood pages, and "where to go for..." landing pages
   - generic icons-plus-image modules like `slow`, `sunset`, `rainy-day`, `fireplace`, `long-lunch`

4. **Experimental cover concepts during pre-production**
   - used internally to test layout, hierarchy and crop pressure before real imagery is sourced

5. **Brand textures and abstract visual system assets**
   - grainy water texture
   - vine-leaf shadow on linen
   - moody window light on tablecloth
   - cellar-door glass reflections

### 3.2 Not approved

1. **Specific venue pages**
   - never on Jackalope, Tedesca, Montalto, Laura, etc.

2. **Specific place pages**
   - never on Red Hill, Sorrento, Cape Schanck, Point Nepean, Flinders, etc.

3. **Experience pages where factual geography matters**
   - never as the primary image for Bushrangers Bay Walk, Sorrento Back Beach, Point Nepean Fort Walk, etc.

4. **Any image implying a documented person, staff member, chef, winemaker, or guest**
   - especially not portraits

5. **Any surface where a reader expects truth-claims from the image**
   - booking surfaces
   - venue cards when the card reads like a review
   - map pins or map previews
   - event listings

6. **Anything passed off as photography**
   - no photorealistic "look how the place really is" usage

## 4. Style direction

### 4.1 The visual ambition

The correct visual reference is not "better AI travel photo."

It is:
- a magazine art director commissioning a still-life or atmospheric plate
- a quiet travel quarterly with restraint
- an editorial supplement that uses illustration to hold tone between real photographs

The output should sit somewhere between:
- **filmic still photography**
- **soft editorial illustration**
- **travel-magazine mood plate**

It should never feel like:
- glossy ad creative
- hyper-real luxury property marketing
- Midjourney maximalism
- Pinterest aspirational fluff
- fantasy concept art

### 4.2 The emotional register

Peninsula Insider is not selling adrenaline, tropical glamour, or escape-porn excess. The emotional tone is:

- slow
- local
- grounded
- autumnal
- wine-country confident
- lightly romantic, not gushy
- premium, but never flashy
- thoughtful enough to feel written, not marketed

### 4.3 The image should feel like

- the hour before lunch, not the afterparty
- a room someone actually sat in, even if no one is present
- a place observed, not staged for a campaign
- soft salt air, fog, linen, timber, stone, glass, vines
- restraint and specific taste

### 4.4 The image should not feel like

- a destination ad
- luxury brochure CGI
- over-rendered fine-art wallpaper
- a tech demo
- a fake photograph trying too hard to pass as real

## 5. Palette and material language

The generative work must harmonise with the existing V2 design system: warm neutrals, wine-country earths, sea-fog softness, editorial contrast.

### 5.1 Core palette directions

- **Warm cream / linen**
- **Weathered stone**
- **Olive leaf / muted eucalyptus**
- **Pinot skin / oxblood / deep burgundy**
- **Sea fog grey-blue**
- **Charcoal timber / blackened steel**
- **Sand / oat / faded terracotta**

### 5.2 Materials to privilege

- linen
- timber
- stone
- ceramic
- hand-blown glass
- wet basalt
- matte plaster
- vine leaves
- paper stock
- candlelight reflections
- fogged windows

### 5.3 Materials to avoid

- chrome-heavy luxury finishes
- neon
- acrylic furniture
- tropical-resort turquoise
- bright synthetic fabrics
- glossy black marble excess
- CGI-perfect reflections

## 6. Composition rules

The generative system should follow a strict compositional grammar so the output family feels cohesive.

### 6.1 Preferred framing

- medium-wide editorial stills
- close textures with shallow depth of field
- negative space for text overlays
- asymmetrical compositions
- subject off-centre
- one clear focal area
- low visual clutter

### 6.2 Preferred camera logic

Use language that implies believable photographic behavior without trying to fake a literal camera capture.

- eye-level or slightly low angle
- natural window light
- overcast outdoor light
- late-afternoon diffused light
- shallow but plausible depth of field
- 35mm / 50mm / 85mm feel
- soft grain, not digital sharpness

### 6.3 Explicitly avoid

- top-down flatlays unless used very sparingly for pantry/produce concepts
- ultra-wide heroic lenses
- drone perspectives
- impossible architecture
- too many objects in frame
- hard flash
- perfect symmetry unless the brief explicitly wants formal hotel architecture

## 7. Surface-specific style rules

### 7.1 Newsletter / signup blocks
Use abstract atmosphere, not scenes. Think linen shadow, wine-glass rim, sea-fog gradient through window light.

**Target effect:** elegant, quiet, background-supporting.

### 7.2 Journal package covers
Use low-claim imagery: table edge, vineyard mist, coastal air, warm interior detail.

**Target effect:** a cover that feels authored, not literal.

### 7.3 Itinerary pages
Use slightly more narrative atmosphere, but still non-specific.

Examples:
- a breakfast table with newspaper and coffee
- a wet road through vines in fog
- candlelight on a table after dusk
- dune grass and sea haze

### 7.4 Intent pages (`long-lunch`, `rainy-day`, `wellness`, `sunset`)
These are the strongest fit for generative support art.

Examples:
- **long-lunch:** linen, wine glass, late light, bread, ceramic, soft shadows
- **rainy-day:** fogged windows, coffee steam, dark timber, paperback, muted grey light
- **wellness:** warm stone, steam, still water, shadowed architecture
- **sunset:** not the sunset itself, but the glow on walls, glass, and table edges

## 8. Labeling and editorial honesty

Every generative asset must be visibly and structurally marked as illustrative.

### 8.1 On-page treatment

At minimum one of these must be true:

- a caption includes `Illustration for Peninsula Insider`
- a hover state / metadata badge says `Illustrative image`
- the page template visually distinguishes illustrative art from documentary photography

### 8.2 Schema recommendation

Extend `imageLicense` in `next/src/content/config.ts` with:

- `generative-illustrative`

Optional additional fields:

```ts
illustrative: z.boolean().default(false)
generationNotes: z.string().optional()
promptRef: z.string().optional()
```

### 8.3 Metadata discipline

Each generated image should have a sidecar metadata record containing:

- prompt
- model / tool used
- generation date
- editor who approved it
- intended slot
- explicit note that it is illustrative, not documentary

## 9. The prompt framework

Prompts should be constructed from five blocks:

1. **Scene** — what exists in frame
2. **Light** — time, weather, softness, reflections
3. **Material palette** — stone, linen, timber, glass, sea fog, vines
4. **Composition** — framing, negative space, focal point
5. **Constraints** — no people, no text, no logos, no hyperrealism, no brochure look

### 9.1 Master prompt formula

```text
[SCENE], in the visual tone of a premium independent travel editorial,
slow and understated, Mornington Peninsula-adjacent mood,
[LIGHT], [MATERIAL PALETTE], [COMPOSITION],
soft filmic grain, restrained colour, naturalistic texture,
illustrative rather than documentary,
no people as main subject, no logos, no signage, no text,
no glossy advertising look, no HDR, no hyperreal luxury brochure styling.
```

### 9.2 Negative-prompt formula

```text
avoid: stock-photo smiles, influencer pose, fake luxury ad aesthetic,
overprocessed sky, drone shot, heavy saturation, hyperreal CGI,
perfect symmetry, cluttered composition, tropical colour palette,
visible branding, artificial text, unusable anatomy, surreal distortions
```

## 11. Prompt modifiers

These are the modular dials that can be added to the templates.

### 11.1 Mood modifiers

- `more autumnal`
- `quieter and more editorial`
- `less romantic, more grounded`
- `slightly more rustic`
- `more wine-country than seaside`
- `more sea-fog than sunshine`
- `more negative space for typography`

### 11.2 Composition modifiers

- `leave clear space in the upper right for headline`
- `closer crop, more tactile material detail`
- `wider frame, fewer objects`
- `reduce focal subjects to one dominant form`
- `introduce soft foreground blur`

### 11.3 Palette modifiers

- `reduce saturation by 20 percent`
- `shift toward oat, olive, burgundy and fog-grey`
- `remove bright blues and bright greens`
- `warmer highlights, cooler shadows`

### 11.4 Anti-cliché modifiers

- `avoid resort imagery`
- `avoid staged lifestyle photography`
- `avoid social-media travel aesthetic`
- `avoid sunset spectacle`
- `avoid Instagram brunch styling`

## 12. Editorial acceptance test for generative outputs

A generated image is acceptable only if it passes all seven checks below.

1. **It reads as illustration, not fake reportage.**
2. **It supports the page without making a factual claim.**
3. **It matches the Peninsula Insider palette and tone.**
4. **It avoids AI telltales** — uncanny objects, broken glassware, warped cutlery, impossible architecture, texture repetition.
5. **It carries compositional restraint.**
6. **It would still feel premium when cropped for mobile.**
7. **It is clearly stronger than the existing CSS placeholder.**

If it fails any one of these, do not use it. The fog placeholder stays.

## 13. Recommended production workflow for future use

This is deliberately lightweight.

### Step 1 — brief the slot
Define:
- page / component
- whether the image is illustrative or documentary
- exact allowed claim level
- crop ratio
- headline overlay needs

### Step 2 — generate a batch
- 4–8 variants max
- one template, one modifier change at a time
- do not open 20 tabs and wander into style drift

### Step 3 — hard review
Reject any output with:
- photoreal fake-documentary ambiguity
- AI artifacts
- wrong palette
- cliché energy
- excessive prettiness without editorial weight

### Step 4 — approve one
Same spirit as the two-step workflow in the image approval document, but shorter:
- shortlist
- editor signs one
- log prompt + metadata
- export and label as illustrative

### Step 5 — periodic purge
Every quarter, review all generative assets. Remove any that:
- now have real-photography replacements
- feel too AI-coded in hindsight
- no longer match the design system

Generative images should have a shorter shelf life than sourced photography.

## 15. Where this should never become a crutch

Three failure patterns to watch for:

### 15.1 The "fake place" trap
A generated vineyard, dining room or hotel suite gets used on a page about a real venue, and the distinction between illustration and documentation blurs.

**Response:** never permit it.

### 15.2 The "brand wallpaper" trap
The site becomes full of tasteful but generic mood art, and starts feeling like a premium template rather than a local guide.

**Response:** use sparingly and replace with real photography as soon as practical.

### 15.3 The "better than nothing" trap
A merely okay generated image gets approved because the placeholder looked empty.

**Response:** the placeholder wins unless the generated output is genuinely stronger.

---

## 16. Recommendation

Use generative imagery **only as a tightly governed illustration layer**, not as a photography layer.

Recommended stance:

- **Approve for:** atmospheric support art, utility-page covers, newsletter textures, editorial package mood plates.
- **Do not approve for:** venue pages, place pages, experience pages, or any surface that implies documentary accuracy.
- **Label clearly.**
- **Keep the volume low.**
- **Replace with real photography over time.**

If used with this discipline, generative imagery can help Peninsula Insider feel authored and premium during the bridge phase. If used loosely, it will make the site feel fake faster than stock photography would.

That tradeoff matters. The bar should stay high.
